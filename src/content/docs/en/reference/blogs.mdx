## Write a Blog Post!

<Aside type="tip" title="A request after participating">
For this event, we ask that you **write a blog post related to the event** after participating. Once you have written one, share it on X or elsewhere with the `#ClaudeCommunityOsaka` hashtag, or let the organizers know. We will feature it on this page.
</Aside>

For anyone thinking "I don't know what to write about," here are some ready-to-use angles.

- **"I tried it" posts** — a reproduction report of today's hands-on session. The places you got stuck, the errors you hit, and how you solved them are the most valuable information of all
- **Comparison / analysis posts** — "Writing your own agent loop with the Messages API vs. Managed Agents," or "When to use Claude Code / the Agent SDK vs. Managed Agents"
- **Applied posts** — extensions of the hands-on session. Ideas grounded in your own life or work are strong: "an agent that posts research results to Slack," "an agent that builds a weekly report from my own business data," and so on
- **Deep-dive posts** — pick one feature we didn't cover today and put it to the test: custom tools, MCP connectors, file inputs (PDFs and images), outcome definitions, and more
